pyio_collections module¶

This module defines the custom collections necessary for the pyio package.

The custom collection, SortedDict, composites behavior of a list and dict, with the added behaior that the elements remain sorted based on a key value that each member of the collection is required to possess.

This module also defines a Base, and derived, class(es) which provide filtered, transpose-like viewing of multiple named members of a derived class when used in combination with a SortedDict.

Example

An example use of a SortedDict is as follows, assuming each element loaded into the SortedDict instance provides an attribute, key, of type float:

series = SortedDict(iter)

An example use of attaching traspose-like viewing to a collection class using the TransposableAsArray class:

series = type('MyObject', (TransposeAsArray, ...), {})(...)

class pyioflash.pyio_collections.SortedDict(iterable)¶

SortedDict is a custom collection implmenting sequence and map like behaviors.

SortedDict is a mutable collection of unique items (that must implement an item.key), providing the following functionality: (1) Sequence-like fast indexable access, (2) Dictionary-like fast mappable access, (3) Collection remains ordered (invariant) based on each element’s key value, (4) Insert/remove (slow), append (fast if sorted), pop (fast) operations, (5) Collection can only contain elements with a unique key (i.e., element.key).

High Level Usage

series = SortedDict(iter)

Creates a collection from iter

series = SortedDict.from_sorted(iter)

Creates a collection from iter; assuming iter is sorted

Common Sequence/Mapping behavior

clear()

Removes all the items from the collection

copy()

Return a shallow copy of the collection

sc = sa + sb

Returns the concatenation of rhs collections

sa += sb

Extends the collection with the contents of sb

sa == sb

Returns logical (equal) comparison of collection’s items; also implements analogous <, <=, !=, >=, and >

Common Sequence/Mapping behavior

append(x)

Add an item to the end of the collection

extend(x)

Extends collection with the contents of t

insert(i, x)

Inserts x into collection at the index given by i

pop([i])

Returns the item at i and removes it from the collection; default i = -1

sort()

Forces a sort of the collection’s items; otherwise defered until the last possible moment

Map-like behavior

get(key[, default])

Return the value for key if key is in the dictionary, else default;

items()

Return an iterator of the collection’s items ((key, value) pairs)

keys()

Returns an iterator of the collection’s keys

pop([key, default])

Returns the key and removes it from the collection, if present, else defualt; if no key, default is last element

popitem()

Remove and return a (key, value) pair from collection; returned in LIFO order

setdefault(default)

If default.key is in collection, return value; otherwise insert/return default

update([other])

Update the collection with the key/value pairs from other; overwriting keys

values()

Returns an iterator of the collections values

Currently the only key type supported is float. This is not a fundemental limitation, however, and an implmentation will be provided in a future update to generalize the key type.

Warning

type(element.key) must be a float

Additional notes w.r.t. access using the [] operator

sd[index] = value

Assignment will overwrite item @ index and sort if necessary; exeption if value.key nonunique and series[index].key != value.key

sd[key] = value

Assignment will insert or overwrite existing item w/ equal key

sd[low : high : set]
(1) Works (nearly) identical to list if args are integers, and
(2) returns a consistant slice matching a closed interval
if args are floats, set must be int

Note

Finally, it is again noted that while the sorted property of SortedDict is invariant; the sorting operation is defered until the last possible moment (e.g., access, index based assignment)

Variables

  • _data – Internal list storing the elementes of the SortedDict

  • _keys – Internal dictionary storing the key value and list index pairs

  • _is_valid – Internal state-flag used to implement just-in-time sorting

class pyioflash.pyio_collections.TransposableAsArray¶

Wrapper class used to provide filtered,transpose-like viewing of multiple named members of a derived collection-like class, wrapped in numpy arrays; super().self must be iterable.

An instance is created with the following syntax:

ta = type('MyObject', (TransposeAsArray, ...), {})(...)

The composited object may be used as before with the following exceptions

Extended behavior as follows

ta[str, …]
Returns a list-like object containing numpy arrays along the key-axis
for each (outer list); matching the args to members of collection
elements, operation is approxamatly a filtered, transpose with wrapping

ta[str, …][:, …]
The returned object has the added property that slicing may be applied
to each inner numpy array elements simultaniously with the [] notation
